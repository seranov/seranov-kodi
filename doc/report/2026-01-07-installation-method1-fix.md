# Исправление Способа 1 установки репозитория

**Дата:** 2026-01-07  
**Тип:** Исправление документации

## Проблема

Пользователь сообщил, что при попытке установки репозитория через **Способ 1** (File Manager с URL GitHub Pages), после создания источника "Seranov Repo" из `https://seranov.github.io/seranov-kodi/`, в списке файлов отображается пустой каталог. Невозможно выбрать `repository.seranov/repository.seranov-1.0.0.zip`.

### Причина

GitHub Pages не предоставляет автоматический листинг директорий для просмотра файлов через File Manager в Kodi. Когда Kodi обращается к URL GitHub Pages, он получает HTML-страницу вместо структуры файлов, что приводит к пустому списку.

## Решение

Изменен **Способ 1** в документации, чтобы использовать прямую установку ZIP-файла через URL вместо добавления источника в File Manager.

### Изменения в документации

#### До изменений:
```markdown
## Способ 1: Установка через URL GitHub Pages (Самый простой)

1. Откройте Kodi
2. Перейдите в **Настройки** → **File manager**
3. Нажмите **Add source**
4. Введите URL: https://seranov.github.io/seranov-kodi/
5. Введите имя для источника (например, `Seranov Repo`)
6. В меню установки выберите `Seranov Repo`
7. Выберите `repository.seranov/repository.seranov-1.0.0.zip`
```

#### После изменений:
```markdown
## Способ 1: Установка через прямой URL (Самый простой)

1. Разрешите установку из неизвестных источников
2. Перейдите в **Настройки** → **Дополнения** → **Установить из файла zip**
3. Выберите **<None>** или введите путь напрямую
4. Введите URL: https://github.com/seranov/seranov-kodi/raw/main/repo/repository.seranov/repository.seranov-1.0.0.zip
5. Нажмите **OK**
6. Дождитесь уведомления "Дополнение установлено"
```

### Преимущества нового подхода

1. ✅ **Работает без дополнительных настроек** - использует прямую ссылку на ZIP-файл
2. ✅ **Меньше шагов** - не требуется создавать источник в File Manager
3. ✅ **Универсальность** - работает с любым хостингом (GitHub Raw, GitHub Pages, и т.д.)
4. ✅ **Надежность** - не зависит от настроек автоматического листинга директорий

### Альтернативные URL

Документация теперь предоставляет два варианта URL:

1. **GitHub Raw (всегда работает):**
   ```
   https://github.com/seranov/seranov-kodi/raw/main/repo/repository.seranov/repository.seranov-1.0.0.zip
   ```

2. **GitHub Pages (если активирован):**
   ```
   https://seranov.github.io/seranov-kodi/repository.seranov/repository.seranov-1.0.0.zip
   ```

## Дополнительные рекомендации

### Для будущих версий

Если требуется сохранить возможность установки через File Manager, необходимо:

1. **Создать index.html** в корне репозитория с автоматическим листингом файлов
2. **Настроить .htaccess** для Apache или эквивалентную конфигурацию для других серверов
3. **Использовать специальные GitHub Actions** для генерации файлового листинга

Однако текущий подход (прямая ссылка на ZIP) является более простым и надежным решением.
